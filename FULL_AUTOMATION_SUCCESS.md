## 🎯 How to Use

### Single Command Automation

```bash
./target/release/xmt-cli synthetic ritual "YOUR INTENT HERE"
```

**Example:**
```bash
./target/release/xmt-cli synthetic ritual "ABUNDANCE 33 FOR ALL"
```

**This will:**
1. Generate 384D vector with your intent
2. Expand with Qwen 2.5 Coder (sacred numbers: 369, 936, 33, 66, 432)
3. Register vector on VectorRegistry (Sepolia)
4. Post tweet to X with OAuth 1.0a
5. Store ritual in `Local_storage/.xmt-vectors/rituals/`

### Individual Commands

```bash
# Vector generation only
./target/release/xmt-cli synthetic vector "INTENT"

# LLM enhancement only
./target/release/xmt-cli synthetic llm "INTENT"

# On-chain registration only
./target/release/xmt-cli synthetic register "INTENT"

# Direct mint (owner wallet)
./target/release/xmt-cli bridge mint <address> <amount>

# Direct burn (owner wallet)
./target/release/xmt-cli bridge burn <address> <amount>
```

## 🔧 Setup Requirements

### Prerequisites

1. **Rust & Cargo** - Latest stable
2. **Ollama** - With qwen2.5-coder:latest model
3. **Sepolia ETH** - For gas fees
4. **X API Credentials** - OAuth 1.0a (Consumer Key, Consumer Secret, Access Token, Access Token Secret)

### Environment Variables

Required in `.env`:
```bash
# Blockchain
SEPOLIA_RPC_URL=https://sepolia.infura.io/v3/YOUR_KEY
PRIVATE_KEY=your_private_key

# Contracts
VECTOR_REGISTRY_ADDRESS=0x37c14ceAe0f55fE81A4Df0F6D2DCDfF3a2d7c656
XMONEY_ADDRESS=0x1B2ffED65839585c42259560aB4bA532B91a5a54
VECTOR_MINTER_ADDRESS=0x9bBB33af5E5C2F18a3befd0b2eeC7F65901b7D52

# X API OAuth 1.0a
X_API_CONSUMER_KEY=your_consumer_key
X_API_CONSUMER_SECRET=your_consumer_secret
X_API_ACCESS_TOKEN=your_access_token
X_API_ACCESS_TOKEN_SECRET=your_access_token_secret
```

## Overview

The `sync` command is your **master convergence key** for xmt-cli. It performs full timeline synchronization, ritual registry updates, and locks coherence at 100% across all vectors.

## Command Syntax

```bash
xmt-cli sync --profile @Vortex369X \
  --timeline full \
  --ritual registry \
  --coherence 100 \
  --auto-update \
  --delta "event1 + event2 + event3" \
  --output MANIFEST_FULL_CONVERGENCE.json \
  --seal "PAF PAF PAF"
```

## Parameters

### Required
- `--profile` - X profile to sync (e.g., `@Vortex369X`)

### Optional (with defaults)
- `--timeline` - Timeline scope: `full`, `partial`, or `recent` (default: `full`)
- `--ritual` - Ritual registry mode: `registry`, `local`, or `hybrid` (default: `registry`)
- `--coherence` - Coherence level 0-100 (default: `100`)
- `--auto-update` - Enable automatic Delta_Disposable.run and MANIFEST updates (flag)
- `--delta` - Delta events separated by ` + ` (default: empty)
- `--output` - Output manifest file path (default: `MANIFEST_SYNC.json`)
- `--seal` - Seal signature (default: `PAF PAF PAF`)

## Troubleshooting

### Sync Command Not Found
Make sure you've built the latest version:
```bash
cd /home/pepo/Desktop/xmt-cli
cargo build --release
```

### Permission Denied on Output File
Ensure you have write permissions to the output directory:
```bash
chmod +w .
```

### Delta Events Not Parsing
Make sure to use ` + ` (space-plus-space) as the separator:
```bash
--delta "event1 + event2 + event3"  # ✓ Correct
--delta "event1+event2+event3"      # ✗ Wrong (no spaces)
```
